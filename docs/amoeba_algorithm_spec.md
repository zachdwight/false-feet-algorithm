# Amoeba Algorithm: Formal Specification
## Adaptive Multi-Branch Reasoning with Self-Supervised Convergence Learning

---

## 1. PROBLEM DEFINITION

**Goal:** Find the best answer in a dataset through adaptive, parallel reasoning paths that compete for resources and converge on a shared solution.

**Inputs:**
- Query: $q$ (user question or problem)
- Dataset: $\mathcal{D} = \{d_1, d_2, ..., d_m\}$ (documents/data points)
- Ground truth (for training): $y^*$ (correct answer)

**Output:**
- Answer: $a$ (final answer)
- Confidence: $\sigma$ (measure of convergence robustness)
- Reasoning trace: $\tau$ (optional, for interpretability)

**Key Properties:**
1. Multiple branches explore independently
2. Branches share retrieved data (pooled evidence)
3. Weak branches get more resources to catch up
4. Branches naturally starve if not improving
5. Convergence = multiple paths reach same answer despite resource disparity

---

## 2. ARCHITECTURE COMPONENTS

### 2.1 Core Components

#### A. Branch (Reasoning Path)
A branch represents one hypothesis/reasoning trajectory.

**State:**
```
Branch_i = {
    reasoning_path: τ_i = [step_1, step_2, ..., step_t],
    current_answer: a_i,
    score: σ_i ∈ [0, 1],
    proposal_slots: n_i ∈ ℤ⁺,
    improvement_counter: c_i ∈ ℤ⁺,
    status: active | starving | dead
}
```

**Definition:**
- `reasoning_path`: Sequence of intermediate reasoning steps generated by branch
- `current_answer`: Tentative answer based on shared data pool at time t
- `score`: Output of evaluator E(τ_i, 𝒫_t)
- `proposal_slots`: Number of documents this branch can propose (allocated inversely by rank)
- `improvement_counter`: Rounds since last score improvement
- `status`: Active (improving), Starving (no improvement), Dead (removed)

#### B. Shared Data Pool
Evidence that all branches draw from.

**State:**
```
𝒫_t = {doc_1, doc_2, ..., doc_k} at time t
|𝒫_t| ≤ B_pool (hard budget constraint)
```

**Properties:**
- All branches see identical 𝒫_t
- Grows via proposals from branches (budgeted)
- Immutable once added (no removal)
- Shared context improves all branches equally, but reasoning diverges

#### C. Evaluator
Neural network that scores reasoning quality.

**Specification:**
```
E: (reasoning_path τ_i, shared_pool 𝒫_t) → score σ ∈ [0, 1]

E(τ_i, 𝒫_t) = neural_network(
    encode(τ_i),        # Reasoning coherence encoding
    encode(𝒫_t),        # Shared evidence encoding
    attention(τ_i, 𝒫_t) # Cross-attention: reasoning ↔ evidence
)
```

**Training:** Self-supervised via convergence outcomes

#### D. Retriever
Component that proposes documents to add to shared pool.

**Specification:**
```
R: (query q, branch_i reasoning τ_i, current_pool 𝒫_t) → proposed_docs [d₁, d₂, ...]

Options:
- Neural: Dense retriever (embedding-based)
- Symbolic: BM25 or keyword matching
- Hybrid: Dense + sparse
```

## 3. ALGORITHM SPECIFICATION

### 3.1 Main Loop: AMOEBA-SEARCH

```
Algorithm: AMOEBA_SEARCH(query q, dataset 𝒟, evaluator E, retriever R)

Input:
    q: query
    𝒟: dataset
    E: evaluator (trained or warm-start heuristic)
    R: retriever
    
Parameters:
    N_init: initial number of branches (e.g., 5)
    N_max: max branches (e.g., 10)
    B_pool: shared pool budget (e.g., 50 documents)
    B_round: per-round proposal budget (e.g., 5 slots)
    T_max: max iterations (e.g., 20)
    ε_improve: improvement threshold (e.g., 0.01)
    ε_starve: starvation threshold (e.g., 3 rounds)
    
Output:
    answer: converged answer
    confidence: convergence robustness
    trace: (optional) full search trace

─────────────────────────────────────────────

1. INITIALIZATION
   branches = spawn_branches(N_init, q)
   𝒫 = {}  # empty shared pool
   t = 0

2. MAIN LOOP: while t < T_max and not converged(branches):
   
   t ← t + 1
   
   ─────── PHASE A: PARALLEL REASONING ───────
   
   3. for each branch_i in branches (in parallel):
      
      4. Generate reasoning step:
         step_t = branch_i.reason(q, 𝒫)
         τ_i ← τ_i + [step_t]
      
      5. Generate intermediate answer:
         a_i ← extract_answer(τ_i, 𝒫)
   
   ─────── PHASE B: EVALUATOR SCORING ───────
   
   6. for each branch_i in branches:
      σ_i ← E(τ_i, 𝒫)  # score reasoning + pool
      
      # Track improvement
      if σ_i > σ_i_prev + ε_improve:
         branch_i.improvement_counter ← 0
      else:
         branch_i.improvement_counter ← branch_i.improvement_counter + 1
   
   ─────── PHASE C: RESOURCE ALLOCATION ───────
   
   7. Rank branches by score: ℛ = RANK(branches by σ)
   
   8. Allocate proposal slots (inverse by rank):
      for i in range(len(ℛ)):
         rank = i + 1
         # Strongest branch (rank 1) gets fewer, weakest gets more
         branch_rank_i.proposal_slots ← max(1, ⌊B_round × (N - rank + 1) / N⌋)
   
   9. Starvation update (progressive death):
      for each branch_i in branches:
         if branch_i.improvement_counter >= ε_starve:
            branch_i.proposal_slots ← max(0, branch_i.proposal_slots - 1)
            branch_i.status ← "starving"
   
   ─────── PHASE D: RETRIEVAL & POOLING ───────
   
   10. Collect proposals:
       proposals = []
       for each branch_i in branches:
          if branch_i.proposal_slots > 0:
             proposed = R(q, τ_i, 𝒫, limit=branch_i.proposal_slots)
             proposals ← proposals + proposed
   
   11. Update shared pool (budgeted):
       # Remove duplicates and rank by relevance
       proposals_dedup = DEDUPLICATE(proposals)
       proposals_ranked = RANK_BY_RELEVANCE(proposals_dedup, q, 𝒫)
       
       # Add to pool up to budget
       slots_available = B_pool - |𝒫|
       new_docs = proposals_ranked[0:slots_available]
       𝒫 ← 𝒫 ∪ new_docs
   
   ─────── PHASE E: ADAPTIVE BRANCHING ───────
   
   12. Diversity check (optional):
       if DIVERSITY(branches) < threshold and |branches| < N_max:
          # Spawn new branch with perturbed reasoning strategy
          new_branch = spawn_branch(q, diversity_mode=True)
          branches ← branches + [new_branch]
   
   ─────── PHASE F: CULLING ───────
   
   13. Remove dead branches:
       branches ← [b for b in branches if b.proposal_slots > 0 or b.status == "active"]
   
   14. CONVERGENCE CHECK
       converged = CHECK_CONVERGENCE(branches, 𝒫)
       if converged:
          break

3. POST-SEARCH: EXTRACT ANSWER

   15. Survivors = [b for b in branches if b.status == "active"]
   
   16. if Survivors is empty:
          answer ← best_answer_overall
          confidence ← LOW
       else:
          answer ← CONSENSUS(Survivors)  # Most common answer
          confidence ← COMPUTE_CONFIDENCE(Survivors)

17. RETURN (answer, confidence, trace)

─────────────────────────────────────────────
```

### 3.2 Helper Functions

#### SPAWN_BRANCHES(N, q)
```
Initialize N independent branches with different reasoning strategies.

for i in 1 to N:
    branch_i.reasoning_path ← []
    branch_i.current_answer ← ∅
    branch_i.score ← 0.5  # neutral start
    branch_i.proposal_slots ← B_round / N  # equal start
    branch_i.improvement_counter ← 0
    branch_i.status ← "active"
    
    # Optional: seed with different strategies
    branch_i.strategy ← SAMPLE_STRATEGY()  # e.g., {"conservative", "exploratory", "direct"}

return [branch_1, ..., branch_N]
```

#### CHECK_CONVERGENCE(branches, 𝒫)
```
All surviving branches reach same answer.

answers = [b.current_answer for b in branches if b.status != "dead"]

if all answers equal each other:
    return TRUE
else:
    return FALSE
```

#### COMPUTE_CONFIDENCE(survivors)
```
Measure robustness of convergence.

# Factor 1: How many branches agree?
agreement_score = len(survivors) / N_init
# More branches surviving despite resource disparity = stronger signal

# Factor 2: How long until convergence?
rounds_to_convergence = t / T_max
# Faster convergence = higher confidence (less uncertainty)

# Factor 3: Score disparity
score_variance = VAR([b.score for b in survivors])
# Low variance among survivors = all branches were convinced

confidence = 0.4 * agreement_score + 0.3 * (1 - rounds_to_convergence) - 0.3 * score_variance

return CLIP(confidence, 0, 1)
```

#### RANK_BY_RELEVANCE(docs, q, 𝒫)
```
Score documents by relevance to query and existing pool.

for each doc in docs:
    relevance_to_query = SIMILARITY(embed(doc), embed(q))
    redundancy_to_pool = max(SIMILARITY(embed(doc), embed(d)) for d in 𝒫)
    
    score = α * relevance_to_query - β * redundancy_to_pool
    # Favor relevant docs, penalize redundancy

return SORT(docs by score, descending)
```

## 4. TRAINING SPECIFICATION

### 4.1 Self-Supervised Learning via Convergence

**Training Objective:** Learn evaluator E to score branches that lead to correct convergence.

#### 4.1.1 Warm Start (Phase 0)
Before self-supervised learning, bootstrap with heuristic evaluator.

```
Heuristic Evaluator:
  σ_heuristic(τ, 𝒫) = w1 * coherence(τ) 
                     + w2 * relevance(τ, q) 
                     + w3 * consistency(τ, 𝒫)

where:
  coherence(τ): Do reasoning steps follow logically?
                [0,1] computed from step embeddings & attention
  
  relevance(τ, q): Do retrieved docs relate to query?
                   SIM(embed(τ), embed(q))
  
  consistency(τ, 𝒫): Does reasoning contradict pool?
                     1 - max(CONFLICT(step, doc) for step, doc)

Weights: w1=0.5, w2=0.3, w3=0.2 (adjustable)
```

#### 4.1.2 Self-Supervised Loss

Once warm-start evaluator is working, train neural evaluator end-to-end.

```
Training Data: {(q_i, 𝒟_i, y_i*) for i in training_set}

For each training example:

  1. Run AMOEBA_SEARCH(q_i, 𝒟_i, E_current) until convergence or timeout
  
  2. Collect outcomes:
     correct_survivors = [b ∈ branches_final if b.answer == y_i*]
     incorrect_survivors = [b ∈ branches_final if b.answer ≠ y_i*]
     dead_branches = [b ∉ branches_final]
  
  3. Compute loss:

     LOSS = L_correct + L_incorrect + L_penalty
     
     where:
     
     ────────────────────────────────────────
     
     L_correct = -Σ log(min_score(b)) for b in correct_survivors
     
     Interpretation: Correct branches should have maintained 
                     confidence throughout search
     
     ────────────────────────────────────────
     
     L_incorrect = +Σ log(max_score(b)) for b in incorrect_survivors
     
     Interpretation: Wrong branches should have been scored low
     
     ────────────────────────────────────────
     
     L_penalty = α * len(dead_branches) / N_init 
               + β * budget_waste(proposals)
     
     Interpretation: Penalize inefficient searches (too much dead weight)
     
     ────────────────────────────────────────
     
     LOSS_total = LOSS + λ * ||E||²  (L2 regularization)

  4. Backprop through entire tree search:
     ∇_θ LOSS_total (where θ are evaluator parameters)
  
  5. Update evaluator:
     θ ← θ - η * ∇_θ LOSS_total
```

### 4.2 Training Procedure

```
Algorithm: TRAIN_EVALUATOR

Input:
    training_data: [(q_1, 𝒟_1, y_1*), ..., (q_N, 𝒟_N, y_N*)]
    E_init: warm-start heuristic evaluator
    
Parameters:
    epochs: number of passes (e.g., 5-10)
    batch_size: examples per batch (e.g., 4)
    learning_rate: η (e.g., 1e-4)
    regularization: λ (e.g., 1e-5)

─────────────────────────────────────────────

1. Initialize: E = E_init

2. for epoch in 1..epochs:
   
   3. Shuffle training_data
   
   4. for batch in batches(training_data, batch_size):
      
      5. losses = []
      
      6. for (q, 𝒟, y*) in batch:
         
         # Run search with current evaluator
         7. (answer, trace) = AMOEBA_SEARCH(q, 𝒟, E)
         
         # Extract outcome
         8. correct_surv, incorrect_surv, dead = extract_outcomes(trace, y*)
         
         # Compute loss (see 4.1.2)
         9. loss = compute_loss(correct_surv, incorrect_surv, dead, trace)
         
         10. losses.append(loss)
      
      # Batch gradient step
      11. avg_loss = mean(losses)
      12. ∇ = backprop(avg_loss)
      13. E.update(∇, learning_rate)
      
      # Logging
      14. log(epoch, batch_idx, avg_loss)
   
   # Validation
   15. val_accuracy = eval_on_validation_set(E)
   16. if val_accuracy improved:
           save_checkpoint(E)

17. return E_best

─────────────────────────────────────────────
```

### 4.3 Convergence of Training

**Training terminates when:**
- Validation accuracy plateaus (no improvement for N epochs)
- Max epochs reached
- Computational budget exhausted

**Quality metrics:**
- Convergence accuracy: % of training examples where AMOEBA converges to correct answer
- Efficiency: Average rounds to convergence
- Robustness: Confidence when correct vs. incorrect

## 5. CONVERGENCE CRITERIA & STOPPING RULES

### 5.1 Search Termination

The search loop terminates when ANY of these conditions is met:

```
TERMINATE CONDITION:

(CHECK_CONVERGENCE(branches, 𝒫) == TRUE)
OR
(t >= T_max)
OR
(|branches| == 0)
OR
(|𝒫| >= B_pool AND no improvement for K rounds)
```

### 5.2 Convergence Definition (Formal)

```
Convergence at time t:

∀ branch_i, branch_j ∈ active_branches(t):
    answer(τ_i, 𝒫_t) == answer(τ_j, 𝒫_t)

AND at least one branch must be active (|active_branches| ≥ 1)
```

### 5.3 Robustness Measures

**Convergence robustness:**
```
R = (num_survivors / num_initial) 
  × (1 - variance(scores_of_survivors))
  × (rounds_to_convergence / T_max)^(-1)

Higher R = more robust convergence
```

## 6. COMPLEXITY ANALYSIS

### 6.1 Time Complexity

Per iteration (round t):

```
Reasoning:         O(N * |τ_max|)           N branches, τ_max steps
Evaluation:        O(N * (|τ| + |𝒫|))       Evaluator scoring
Retrieval:         O(K * log(|𝒟|))          K proposals, search dataset
Pooling:           O(K log K + B_pool)      Sort & merge proposals
Culling:           O(N)                      Remove dead branches

Total per round:   O(N * |τ_max| + K log |𝒟|)

Total search:      O(T_max * (N * |τ_max| + K log |𝒟|))
```

Typical values: N=5-10, T_max=10-20, K=3-5, |τ_max|=10
→ **O(1000-5000) evaluations per query**

### 6.2 Space Complexity

```
Branches:          O(N * |τ_max|)           All reasoning paths
Shared pool:       O(B_pool * |d_avg|)      Retrieved documents
Evaluator:         O(parameters)            Neural network weights
Retriever:         O(|𝒟| + embeddings)      Dataset + indices

Total:             O(N * |τ_max| + B_pool * |d_avg| + parameters)

Typical: O(10 KB - 1 MB) per query
```

## 7. HYPERPARAMETER GUIDANCE

| Parameter | Typical Range | Interpretation |
|-----------|---------------|-----------------|
| N_init | 3-10 | Starting branches; more = diversity, cost |
| N_max | 5-15 | Cap on branches; prevents explosion |
| B_pool | 20-100 | Shared data budget; larger = more context, cost |
| B_round | 3-10 | Per-round proposals; affects speed/cost |
| T_max | 10-50 | Max iterations; termination safety |
| ε_improve | 0.001-0.01 | Score improvement threshold |
| ε_starve | 2-5 | Rounds before starvation begins |
| learning_rate | 1e-5 to 1e-3 | Evaluator training rate |
| λ (regularization) | 1e-6 to 1e-4 | Evaluator complexity penalty |

## 8. PSEUDOCODE SUMMARY

```python
# Quick reference implementation sketch

class AmoebaBranch:
    def __init__(self, query):
        self.reasoning_path = []
        self.answer = None
        self.score = 0.5
        self.proposal_slots = 5
        self.improvement_counter = 0
        self.status = "active"
    
    def reason(self, query, shared_pool):
        step = generate_next_step(query, self.reasoning_path, shared_pool)
        self.reasoning_path.append(step)
        self.answer = extract_answer(self.reasoning_path, shared_pool)
        return step

class AmoebaBrain:
    def __init__(self, evaluator, retriever, dataset):
        self.evaluator = evaluator  # Neural network
        self.retriever = retriever  # Document search
        self.dataset = dataset
    
    def search(self, query):
        branches = [AmoebaBranch(query) for _ in range(5)]
        shared_pool = set()
        
        for t in range(20):  # Max iterations
            # Phase A: Reasoning
            for branch in branches:
                branch.reason(query, shared_pool)
            
            # Phase B: Evaluation
            for branch in branches:
                prev_score = branch.score
                branch.score = self.evaluator(branch.reasoning_path, shared_pool)
                if branch.score > prev_score + 0.01:
                    branch.improvement_counter = 0
                else:
                    branch.improvement_counter += 1
            
            # Phase C: Resource allocation
            ranks = sorted(range(len(branches)), 
                          key=lambda i: branches[i].score)
            for rank, idx in enumerate(ranks):
                branches[idx].proposal_slots = (len(branches) - rank)
                if branches[idx].improvement_counter >= 3:
                    branches[idx].proposal_slots = max(0, branches[idx].proposal_slots - 1)
            
            # Phase D: Retrieval & pooling
            proposals = []
            for branch in branches:
                if branch.proposal_slots > 0:
                    docs = self.retriever(query, branch.reasoning_path, 
                                         limit=branch.proposal_slots)
                    proposals.extend(docs)
            
            new_docs = rank_by_relevance(proposals, query, shared_pool)[:5]
            shared_pool.update(new_docs)
            
            # Phase F: Culling
            branches = [b for b in branches if b.proposal_slots > 0 or b.status == "active"]
            
            # Convergence check
            answers = [b.answer for b in branches]
            if len(set(answers)) == 1:
                return answers[0]
        
        return branches[0].answer  # Fallback
```

## 9. KEY INVARIANTS

1. **Shared pool monotonicity:** $|\mathcal{P}_t| \leq |\mathcal{P}_{t+1}|$ (pool only grows)
2. **Resource allocation:** Weaker branches always get ≥ resources than stronger ones (inverse ranking)
3. **Convergence validity:** If convergence detected, all survivors must have identical answers
4. **Starvation:** If improvement_counter ≥ ε_starve, proposal_slots monotonically decrease
5. **Budget constraint:** $|\mathcal{P}_t| \leq B_{pool}$ at all times

---

## 10. FAILURE MODES & MITIGATION

| Failure Mode | Symptom | Mitigation |
|--------------|---------|-----------|
| **Poor evaluator** | All branches score equally; no selection pressure | Warm-start with heuristic; increase epochs |
| **Premature convergence** | Converges to wrong answer | Increase diversity spawning; increase T_max |
| **No convergence** | All branches die before agreement | Increase B_pool; decrease starvation threshold |
| **Computational cost** | Search too slow for real-time | Reduce N_init; reduce B_round; reduce B_pool |
| **Over-retrieval** | Pool fills with noise | Improve relevance ranking; reduce B_pool |
| **Evaluator overfitting** | High training acc, low test acc | Increase regularization λ; use early stopping |

## 11. COMPARISON TO BASELINES

| Aspect | LLM (baseline) | Tree Search | Amoeba |
|--------|---|---|---|
| Search type | Feed-forward generation | Exhaustive branching | Adaptive competition |
| Resource allocation | Uniform (all tokens) | Uniform (all branches) | **Inverse (weakest favored)** |
| Data integration | Cached at start | Re-fetched per branch | **Shared pool (budgeted)** |
| Convergence signal | None (single path) | Coverage | **Agreement (multiple paths)** |
| Training | Supervised (token-level) | Supervised (outcome) | **Self-supervised (convergence)** |
| Efficiency | Fast (feed-forward) | Slow (exhaustive) | **Medium (adaptive)** |
| Robustness | Low (single point of failure) | Medium (if complete) | **High (converged diversity)** |

## APPENDIX B: FUTURE EXTENSIONS

1. **Multi-answer problems:** Branches converge to *multiple* valid answers (modify convergence check)
2. **Soft convergence:** Branches don't need identical answers, just "close enough" (fuzzy matching)
3. **Streaming mode:** New data arrives during search; pool updates continuously
4. **Hierarchical branches:** Branches spawn sub-branches for sub-problems
5. **Explainability:** Trace which data/reasoning led to final answer
6. **Uncertainty quantification:** Confidence scores reflecting model uncertainty + convergence robustness
